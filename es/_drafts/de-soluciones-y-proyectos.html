He decidido dividir el contenido y si llegaron acá buscando cosas técnicas este es el lugar indicado. lang: es alias: /hola-mundo/index.html tags: - meta --- Antes de adentrarnos un poco más en el desarrollo de aplicaciones con C# y Visual (o Xamarin) Studio debemos saber de qué manera estará organizado nuestro código dentro del entorno de programación, y es por eso que decidí escribir este post para tratar de aclarar algunos conceptos que al inicio pueden resultar un tanto confusos.
Para ayudarme un poco, usaré como muestra una app llamada "ToDo" que estoy desarrollando para un peque;o taller que estar[e dando muy porointo.
Volviendo a los conceptos confusos... son: Solution alias *La soluci[on* y Project alias *El proyecto*. Digo que son confusos porque normalmente cuando alguien habla de el desarrollo de un sistema, un sitio web o una app dice "Estoy trabajando en el proyecto X", lo cual, trasladado a Visual Studio puede no ser tan correcto. Pero bueno, echemos un ojo a c[omo En C# se conoce como Solution al contenedor lógico de todo el proyecto sobre el que estamos trabajando. Al decir que es un contenedor lógico quiero decir que en el sistema de archivos del sistema operativo "La soliución" es un archivo que apunta hacia nuestros proyectos más no los contiene. Dicho archivo siempre tiene una extensión .sln.
